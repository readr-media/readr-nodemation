# readr-nodemation

「readr-nodemation」是一個視覺化的工作流建構工具，旨在讓新聞編輯與開發者能透過拖曳方式，串連不同的 AI 模組與 CMS 系統，自動化新聞處理流程。

## 核心功能

- **視覺化流程建構**：使用者可透過拖曳模組、連接節點的方式，直觀地設計出所需的新聞自動化流程。
- **模組化設計**：提供多樣化的功能模組，包含 AI 應用（如自動標籤、自動分類）、CMS 操作（讀取、寫入）及通用工具（如產出報告、匯出結果）。
- **即時監控與測試**：可在畫布上即時查看每個節點的執行狀態（閒置、執行中、成功、失敗），並能針對單一節點進行測試。
- **任務紀錄與管理**：所有執行過的工作流都會被記錄下來，方便使用者隨時查詢歷史紀錄、檢視結果或對失敗的任務進行重跑。
- **模組管理**：提供統一的介面，讓使用者管理所有可用的模組，包含新增、編輯 API 金鑰或刪除模組。

## 主要介面

1.  **Workflow Builder (主畫面)**：
    *   **左側 (Modules Panel)**：顯示所有可用的模組，支援搜尋與分類，可將模組拖曳至畫布。
    *   **中間 (Flow Canvas)**：主要操作區，用於建構與呈現工作流的節點與連線。
    *   **右側 (Config Panel)**：當點選任一節點時，此區會顯示該節點的詳細設定選項。
    *   **上方 (Top Bar)**：全域控制列，包含儲存、執行、導航至其他頁面等主要操作。

2.  **Workflow 列表頁**：
    *   顯示所有已儲存的工作流，提供搜尋與篩選功能。
    *   可從此處進入 Builder 編輯既有流程，或建立新的空白流程。

3.  **執行紀錄 (Task History)**：
    *   條列出所有歷史任務的執行紀錄，包含任務名稱、時間、狀態等。
    *   點擊單筆紀錄可查看詳細的輸入/輸出內容與 MCP JSON 檔案。

4.  **模組管理 (Module Library)**：
    *   集中管理所有可用的模組，包含其 API 設定、版本等資訊。

## 基礎使用者流程

1.  **登入**：從現有的 CMS 登入後，點擊進入「AI 新聞工作台」。
2.  **建構流程**：在 Workflow Builder 主畫面，從左側拖曳所需模組至畫布，並將它們連接起來，設定各節點的參數。
3.  **執行與儲存**：點擊 Top Bar 上的「執行」按鈕來運行整個工作流，或點擊「儲存」以保存當前進度。
4.  **查看結果**：可至「任務紀錄」頁面查看詳細的執行結果與 log。
5.  **管理與維護**：透過「Workflow 列表頁」與「模組管理」頁面，管理已建立的流程與可用的模組。
